13. File yang berisi konfigurasi lokal DNS server untuk meletakkan zona lokal adalah


Jawaban:

 

File yang berisi konfigurasi lokal DNS server untuk meletakkan zona lokal adalah:

 

etc/bind/ named.conf.local

 

Pembahasan:

 

Pada server jaringan komputer berbasis UNIX/LINUX, pengaturan konfgurasi DNS (Domain Name Seer) dan pengaturan jaringan lainya dilakukan dengan BIND (Berkeley Internet Name Domain).

 

BIND (dan variannya seperti BIND9) adalah server DNS yang paling umum digunakan di Internet, khususnya pada sistem operasi bertipe UNIX/LINUX yang secara de facto merupakan standar.

 

Konfigurasi pengaturan DNS pada bind ini dilakukan dalam serangkaian file.

 

File-file konfigurasi DNS yang tersimpan dalam direktori:

 

/etc/bind

 

Direktori atau folder ini tersusun dari beberapa file. File konfigurasi utama dalam folder ini adalah file:

 

/etc/bind/ named.conf.

 

Sedangkan file-file lainya adalah:

 

/usr/sbin/ named = file binary untuk menjalankan bind9

 

/etc/init.d/ bind9 = file init untuk memudahkan start/stop/restart named

 

/etc/bind/ named.conf.options = file untuk mengatur forwarder untuk keperluan cache nameserver

 

/etc/bind/ named.conf.local = file yang isinya adalah konfigurasi lokal dns server, disini kita akan meletakkan zona lokal.

 

/etc/bind/ named.conf.default-zones = file yang isinya default zones (db.local, db.0, db.127, db.255)

15. Bagaimana Cara konfigurasi DNS Server di OS Linux. ?​


Penjelasan:

1.Membuat domain dari server linux dengan menggunakan TLD (Top Level Domain). TLD ini hanya berfungsi pada jaringan lokal saja. Langkahnya adalah dengan mengkonfigurasi file named.conf.local pada direktori /etc/bind

2.Buatlah file forward dengan cara copy file db.local pada direktori /etc/bind.

File forward berfungsi untuk konversi dari DNS ke IP Address. 

3.Buatlah file reserve dengan cara copy file db.127 pada direktori /etc/bind. File reserve berfungsi untuk konversi IP Address ke DNS. 

4. Tambahkan DNS nameserver dari server linux pada file resolv.conf.

5.Test apakah DNS Server tersebut berhasil atau tidak, dengan perintah ping maupun nslookup dari computer Localhost maupun computer client.

36. apa yang dimaksud dengan dns dan apa perbedaan public dns dengan dns default


DNS yaitu Sebuah sistem yang menyimpan informasi tentang nama host ataupun nama domain dalam bentuk basis data tersebar (distributed database) di dalam jaringan komputer.

Jawaban:

DNS atau Domain Name System berfungsi sebagai penerjemah/pengubah dari alamat situs ke bentuk IP address.

Simplenya DNS adalah buku telepon agar perangkat kita dapat menemukan IP address dari suatu situs web dengan lebih cepat.

Contoh DNS

Beberapa DNS publik yang sudah sangat umum yaitu DNS Google (8.8.8.8) dan OpenDNS (208.67.222.222). Banyak penyedia Public DNS lainnya yang memberikan layanan DNS secara gratis seperti Cloudflare, AdGuard, Quad9, Verisign, dll. yang menyediakan keunggulannya masing-masing.

Semua tergantung User dalam memilih ingin menggunakan DNS yang mana.

Perbedaan

DNS Publik atau Public DNS memiliki beberapa keunggulan tersendiri tergantung dari penyedia DNS-nya. Terkadang menggunakan Public DNS jauh lebih menguntungkan dalam beberapa hal dibandingkan menggunakan DNS default bawaan provider.

Contoh Penggunaan

Menggunakan Public DNS seperti Google DNS lebih menguntungkan daripada menggunakan DNS bawaan provider karena server Google sendiri sudah sangat canggih, jauh lebih maju dibanding server yang digunakan provider internet tertentu sehingga dalam beberapa kasus jika kita menggunakan Google DNS atau DNS publik lainnya maka aktivitas browsing kita bisa jauh lebih lancar dan cepat.

Lalu biasanya DNS bawaan dari provider internet kita sudah dimodifikasi agar menampilkan iklan yang mempromosikan produk & layanan mereka. Beberapa pengguna pastinya merasa terganggu dengan iklan-iklan ini. Hal ini tentunya tidak ada jika menggunakan Public DNS dari Google, OpenDNS, Cloudflare, dsb.

Catatan

Yang perlu diperhatikan adalah pastikan untuk selalu menggunakan Public DNS dari penyedia yang memang terpercaya seperti Google, Cloudflare, OpenDNS, dan sejenisnya. Hal ini karena aktivitas browsing dapat dicatat oleh server DNS. Agar tidak disalah-gunakan maka harus menggunakan penyedia Public DNS yang terpercaya.


37. Tuliskan Langkah-langkah menambahkan alamat DNS di Windows agar client bisa mengakses Domain yang sudah dikonfigurasi.​


Berikut adalah langkah-langkah menambahkan alamat DNS di Windows agar client bisa mengakses domain yang sudah dikonfigurasi:

Buka Control Panel pada Windows dan pilih Network and Sharing Center.

Klik pada opsi Change adapter settings.

Klik kanan pada adapter yang sedang digunakan dan pilih Properties.

Pilih Internet Protocol Version 4 (TCP/IPv4) dan klik Properties.

Pilih opsi Use the following DNS server addresses.

Masukkan alamat DNS yang ingin ditambahkan pada Preferred DNS server dan Alternate DNS server.

Klik OK untuk menyimpan perubahan.

Ulangi langkah-langkah ini pada setiap komputer client yang ingin mengakses domain yang sudah dikonfigurasi.

Setelah menambahkan alamat DNS pada komputer client, pengguna dapat mengakses domain yang sudah dikonfigurasi melalui web browser atau aplikasi lainnya yang memerlukan koneksi internet. Pastikan untuk memeriksa kembali setiap konfigurasi DNS pada server dan client untuk memastikan bahwa semuanya sudah benar.
