3. Langkah-langkah penelusuran binary tree dengan cara Out Order adalah a. Print data, telusuri kanan, telusuri kiri b. Telusuri kiri, print data, telusuri kanan c. Print data, telusuri kiri, telusuri kanan d. Tidak ada jawaban yang benar


Jawaban:

Penjelasan:

karna setauku hanya ada :

In-order : kiri, root, kananPre-order : root, kiri, kananPost-order : kiri, kanan, root

Jawaban yang benar untuk menjawab soal tersebut adalah D. Tidak ada jawaban yang benar. Alasannya karena tidak ada cara Out order pada penelusuran binary tree. Penelusuran binary tree hanya mencakup pre order, in order dan post order.

PEMBAHASAN:

Traversal merupakan proses kunjungan dalam pohon dimana setiap simpul hanya dikunjungi tepat satu kali.

Terdapat tiga kegiatan yang ada dalam traversal pohon binar (binary tree) yaitu:

Mengunjungi simpul akar (root)Melakukan traversal subpohon kiriMelakukan traversal subpohon kanan

Selain itu terdapat pula tiga macam traversal pohon, diantaranya:

1.   Traversal Pre-order, caranya:

Mengunjungi simpul akarMelakukan traversal subpohon kiriMelakukan traversal subpohon kanan

2.   Traversal In-order, caranya:

Melakukan traversal subpohon kiriMengunjungi simpul akarMelakukan traversal subpohon kanan

3.   Traversal Post-order, caranya:

Melakukan traversal subpohon kiriMelakukan traversal subpohon kananMengunjungi simpul akar

Tree/pohon adalah struktur data dimana data tersebut tidak linear/non linear yang digunakan terutama untuk merepresentasikan suatu hubungan data yang sifatnya hierarkis antara elemen-elemennya.

Buat lah sebuah 3 fungsi overloading yang memiliki 1 parameter - Jika parameternya int maka outputnya “Tipe data int, hasilnya [parameter * 100]” - Jika parameternya double maka outputnya “Tipe data double, hasilnya [parameter * 1000]” - Jika parameternya string maka outputnya “Tipe data string, nilainya Hello [parameter]” contoh pada fungsi main : int main() { overloadingFunc(10; overloadingFunc(10.0); overloadingFunc("World); return 0; } outputnya : Tipe data int, hasilnya 10000 Tipe data float, hasilnya 1000 Tipe data string, nilainya Hello World


Jawaban dan Penjelasan:

Overloaded Functions

Saya buat 2 versi. Yang pertama, argumen bertipe string menggunakan tipe "string" dari namespace std. Karena ada baris "use namespace std;", maka nama tipenya "string" saja, tidak perlu "std::string".

Versi kedua, argumen bertipe string menggunakan "char*". Pada saat passing argumen ke fungsi, harus menggunakan casting berupa "(char*)", agar tidak ada warning pada saat kompilasi.

Versi 1: tipe data string menggunakan string dari namespace std (std::string)

#include <iostream>

using namespace std;

void overloadingFunc(int i) {

   cout << "Tipe data int, hasilnya " << (100*i) << endl;

}

void overloadingFunc(double d) {

   cout << "Tipe data double, hasilnya " << (1000*d) << endl;

}

void overloadingFunc(string s) {

   cout << "Tipe data string, nilainya Hello " << s << endl;

}

int main() {

   overloadingFunc(10);

   overloadingFunc(10.0);

   overloadingFunc("World");

   return 0;

}

Versi 2: tipe data string menggunakan char*

Bagian di atas ini persis sama seperti di atas.

void overloadingFunc(char* s) {

   cout << "Tipe data string, nilainya Hello " << s << endl;

}

int main() {

   overloadingFunc(10);

   overloadingFunc(10.0);

   overloadingFunc((char*)"World");

   return 0;

}

19. Proses komunikasi data yang termasuk destination system adalah?​


Jawaban:

Proses komunikasi data yang termasuk dalam destination system, yaitu:

1. Pengiriman data: Data dikirim dari sistem asal atau pengirim ke sistem tujuan atau penerima. Ini melibatkan pengkodean data dan kegiatan pengiriman data melalui jaringan atau media komunikasi yang digunakan.

2. Penerimaan data: Sistem tujuan atau penerima menerima data yang dikirimkan oleh sistem asal. Ini melibatkan proses dekode data untuk mengembalikan data ke bentuk aslinya.

3. Penyimpanan atau pengolahan data: Setelah data diterima, sistem tujuan dapat menyimpan atau mengolah data sesuai dengan kebutuhan atau tujuan yang ditentukan.

4. Verifikasi atau validasi data: Sistem tujuan dapat memverifikasi atau memvalidasi data yang diterima untuk memastikan keotentikan dan integritas data.

5. Akses data: Setelah data valid dan diverifikasi, data tersebut dapat diakses oleh pengguna atau sistem lain yang membutuhkannya.

6. Penggunaan data: Data yang diterima oleh sistem tujuan dapat digunakan untuk tujuan tertentu, seperti pemrosesan, analisis, manajemen, atau tindakan lebih lanjut.

7. Penyimpanan jangka panjang: Data yang diproses dan digunakan dapat disimpan dalam sistem tujuan untuk jangka waktu yang lebih lama, jika diperlukan.

Proses-proses ini mencakup langkah-langkah penting dalam komunikasi data dari sistem asal ke sistem tujuan atau destination system.

21. Ukuran Memori dari tipe data int / Integer adalah​


1. Tipe data Integer / Ordinal

Tipe data ini adalah tipe data yang berupa bilangan bulat yang akan digunakan untuk operasi matematika. Ada beberapa tipe data yang termasuk integer, yaitu antara lain:

a. ShortInt : memiliki nilai range -128 sampai 127 (signed 8-bit)

b. SmallInt : memiliki nilai range antara - 32768 sampai dengan 32767 (signed 16-bit)

c. LongInt : memiliki nilai range -2147483648 sampai 2147483647 (signed 32-bit )

d. Int64 : memiliki range -263+1 sampai 263-1 (signed 64-bit)

e. Byte : memiliki nilai range 0 sampai 255 (unsigned 8-bit)

f. Word : memiliki nilai range 0 sampai 65535 (unsigned 16-bit)

g. LongWord : memiliki nilain range 0 sampai 4294967295(unsigned 32-bit )

Pada pemakaian integer type di implemetasi 16 bit maka:

a. Integer : memiliki range antara - 32768 sampai dengan 32767 (signed 16-bit)

b. Cardinal : memiliki range 0 sampai 65535 (unsigned 16-bit)

Pada pemakaian integer type di implemetasi 32 bit maka:

a. Integer : memiliki range antara -2147483648 sampai 2147483647 (signed 32-bit)

b. Cardinal : memiliki range 0 sampai 4294967295 (unsigned 32-bit)
