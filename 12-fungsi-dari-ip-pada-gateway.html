5. Diketahui IP 168.100.10.3/23 ,Coba tentukan IP awal, IP akhir, subnetmask, gateway,ip broadcast, ip akhir dan jumlah host(client)​


Jawaban:

berikut cara menghitung IP

Sebutkan Host, Network ID, Broadcast, Subnet Mask ?

Contoh Soal 1 :

Sebutkan Network ID, Broadcast ID, Subnet Mask dan Host ID pada IP 192.168.100.80/25

Untuk menghitung Host ID. /25 = 32 -25 = 7

2^7 = 128

Berarti Jumlah Host ID = 128 Host

128 host ini nantinya dikurangi 2, yaitu untuk Network ID dan Broadcast

Untuk Menghitung Network ID (Host Pertama)

Network ID pada 192.168.100.80 = 192.168.100.0

Untuk Menghitung Broadcast (Host Terakhir)

Broadcast pada 192.168.100.80/25 = 192.168.100.127

IP Address yang bisa digunakan

192.168.100.1 – 192.168.100.126

Untuk Menghitung Subnet Mask

Subnet Mask pada 192.168.100.80/25 = 256 – 128 = 128

Maka Subnetmasknya adalah = 255.255.255.128

Contoh Soal 2 :

Sebutkan Network ID, Broadcast ID, Subnet Mask dan Host ID pada IP 20.20.0.22/22

Untuk menghitung Host ID. /22 = 32 -22 = 10

2^10 = 1024

Berarti Jumlah Host ID = 1024 Host

1024 host ini nantinya dikurangi 2, yaitu untuk Network ID dan Broadcast

Untuk Menghitung Network ID (Host Pertama)

Network ID pada 20.20.0.22 = 20.20.0.0

Untuk Menghitung Broadcast (Host Terakhir)

1024 / 256 = 4

Broadcast pada 20.20.0.22/22 = 20.20.3.255

IP Address yang bisa digunakan

20.20.0.1 – 20.20.3.254

Untuk Menghitung Subnet Mask

Subnet Mask pada 20.20.0.22/22

2^10 adalah kelas B maka 10-8 = 2 = 2^2 = 4. 256 -4 = 252

Maka Subnetmasknya adalah = 255.255.252.0

Contoh Soal 3 :

Sebutkan Network ID, Broadcast ID, Subnet Mask dan Host ID pada IP 10.10.10.10/24

Untuk menghitung Host ID. /24 = 32 -24 = 8

2^8 = 256

Berarti Jumlah Host ID = 256 Host

256 host ini nantinya dikurangi 2, yaitu untuk Network ID dan Broadcast

Untuk Menghitung Network ID (Host Pertama)

Network ID pada 10.10.10.10/24 = 10.10.10.0

Untuk Menghitung Broadcast (Host Terakhir)

Broadcast pada 10.10.10.10/24 = 10.10.10.255

IP Address yang bisa digunakan

10.10.10.1 – 10.10.10.254

Untuk Menghitung Subnet Mask

Subnet Mask pada 10.10.10/24 = 256 – 256 = 0

Maka Subnetmasknya adalah = 255.255.255.0

Note :

Anda perlu tahu Subnetmask Kelas IP Address

Kelas A = 255.0.0.0

Kelas B = 255.255.0.0

Kelas C = 255.255.255.0

Subnet mask yang sering digunakan biasanya adalah /24 = 255.255.255.0

10. Jika ip gateway 197.16.12.202/29 ip Address yang tidak mungkin di pakai oleh clien adalah


Penjelasan:

Untuk menentukan alamat IP yang tidak mungkin digunakan oleh klien, pertama kita perlu memahami subnet mask yang diberikan. Dalam kasus ini, subnet mask adalah /29, yang berarti terdapat 29 bit yang ditetapkan sebagai jaringan dan sisanya sebagai host.

Dalam alamat IP 197.16.12.202/29, bit-bit jaringan adalah 197.16.12.200, sedangkan bit-bit host adalah 202. Oleh karena itu, alamat IP yang tidak mungkin digunakan oleh klien adalah alamat yang memiliki nilai host yang berbeda dari 202 dalam rentang subnet tersebut.

Berikut adalah daftar alamat IP yang tidak mungkin digunakan oleh klien dalam subnet tersebut:

- 197.16.12.201

- 197.16.12.202 (alamat IP yang digunakan oleh gateway)

- 197.16.12.203

- 197.16.12.204

- 197.16.12.205

- 197.16.12.206

- 197.16.12.207

Semua alamat IP di atas termasuk dalam jaringan yang didefinisikan oleh subnet /29 dan tidak dapat digunakan oleh klien.

Jawaban:

Untuk menentukan alamat IP yang tidak mungkin digunakan oleh client, kita perlu mengetahui subnet mask dari alamat IP gateway tersebut. Dalam hal ini, subnet mask adalah /29, yang berarti terdapat 3 bit host pada jaringan tersebut.

Kita dapat menghitung jumlah alamat IP yang tersedia pada jaringan tersebut dengan menggunakan rumus 2^n - 2, di mana n adalah jumlah bit host pada subnet mask. Dalam hal ini, n = 3, sehingga jumlah alamat IP yang tersedia adalah 2^3 - 2 = 6.

Alamat IP yang tidak mungkin digunakan oleh client adalah alamat IP yang termasuk dalam jangkauan alamat IP yang tersedia pada jaringan tersebut, tetapi tidak dapat digunakan karena telah dialokasikan untuk keperluan lain. Dalam hal ini, alamat IP gateway adalah 197.16.12.202/29, sehingga alamat IP yang tidak mungkin digunakan oleh client adalah:

- 197.16.12.202 (alamat IP gateway)

- 197.16.12.203 (alamat IP pertama pada jaringan)

- 197.16.12.204 (alamat IP kedua pada jaringan)

- 197.16.12.205 (alamat IP ketiga pada jaringan)

- 197.16.12.206 (alamat IP keempat pada jaringan)

- 197.16.12.207 (alamat IP terakhir pada jaringan)

Jadi, alamat IP yang tidak mungkin digunakan oleh client adalah 197.16.12.203, 197.16.12.204, 197.16.12.205, 197.16.12.206, dan 197.16.12.207.

15. Hubungan Ip address dengan gateway


Hubungan Ip address dengan gateway, sebelum membahas lebih lanjut mengenai hubungan IP Address dengan gateway, mari kita bahas secara singkat apa itu IP Adress dan apa itu Gateway. IP Address dapat diartikan sebagai alamat suatu komputer. IP Address berupa kombinasi angka. IP Address sendiri terbagi menjadi beberapa kelas, penggunaan IP Address sendiri berbeda satu dengan yang lainnya. Pemberian IP Address antara satu komputer dengan yang lainnya juga harus berbeda.

Gateway sendiri adalah suatu jalur atau portal yang digunakan komputer pada suatu jaringan komputer, penginputan gateway sendiri juga sama dengan IP Address yaitu kombinasi dari berbagai angka. Adapun hubungan antara IP Address dengan Gateway adalah keduanya saling berkaitan, komputer dengan IP Adress tanpa Gateway tidak akan berfungsi pada jaringan komputer begitu juga sebaliknya. Untuk penggunaan IP Address dan Gateway sendiri sudah terdapat pengaturan yang mengaturnya.

Demikian pembahasan mengenai hubungan antara IP Address dan Gateway, yaitu keduanya saling berkaitan, contoh IP Address komputer A 172.16.253.197 maka Gateway dari komputer tersebut biasanya memiliki angka yang lebih kecil yaitu 172.16.253.195 untuk membaca materi lainnya mengenai IP Address dan materi lainnya mengenai jaringan komputer dapat dibaca pada link berikut

38. Ip gateway yang mungkin digunakan jika ip client adalah 192. 168. 20. 254/30 adalah…


Jika IP client adalah 192.168.20.254/30, maka IP gateway yang mungkin digunakan adalah 192.168.20.253.

IP client adalah alamat IP yang digunakan oleh perangkat klien (misalnya komputer atau smartphone) untuk terhubung ke jaringan. Sedangkan IP gateway adalah alamat IP dari perangkat yang bertugas sebagai penghubung antara perangkat klien dengan jaringan luar.

Dalam kasus ini, IP client menggunakan prefix subnet 30, yang berarti hanya terdapat 2 bit yang digunakan untuk mengidentifikasi host. Jadi, mungkin ada hanya 2 perangkat yang terhubung ke jaringan dengan prefix subnet yang sama, yaitu perangkat dengan alamat IP 192.168.20.254 dan perangkat dengan alamat IP 192.168.20.253. Perangkat dengan alamat IP 192.168.20.253 mungkin merupakan IP gateway yang digunakan oleh perangkat klien.
