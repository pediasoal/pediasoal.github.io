6. PC Arya memiliki alamat IP ADDRESS 192.168.100.1 SUBNETMASK: 255.255.255.0, PC Arga memiliki alamat IP ADDRESS 192.168.1.100 SUBNETMASK: 255.255.255.0. Apakah PC Arga dan Arya bisa terhubung? Apa alasannya?​


PC Arga dan Arya dapat terhubung.
Alasannya secara singkat adalah karena kedua PC tersebut berada dalam sebuah subnet yang sama, yaitu subnet dengan network address 192.168.100.0, dengan catatan:

Kedua PC berada dalam jaringan lokal yang sama, baik terhubung dengan kabel maupun nirkabel.Minimal 1 service terbuka pada kedua PC, atau dibuka oleh administrator, misalnya ICMP (sehingga keterhubungan kedua PC dapat diperiksa dengan ping).

 

Pembahasan

PC Arya memiliki alamat:
IP ADDRESS 192.168.100.1 SUBNETMASK: 255.255.255.0.

Network ID: 192.168.100Oktet pertama = 192, maka jenis jaringan adalah private (jaringan lokal).(192.168.100).00000001 AND (255.255.255).00000000 = 192.168.100.0
⇒ Network address = 192.168.100.0Oktet terakhir subnet mask = 00000000
⇒ Jumlah subnet = 2⁰ = 1 subnet
⇒ Jumlah host per subnet = 2⁸ – 2 = 256 – 2 = 254 hostIP address yang dapat dipergunakan: 192.168.100.1 – 192.168.100.254Broadcaset IP address: 192.168.100.255

Sedangkan PC Arga memiliki alamat:
IP ADDRESS 192.168.1.100 SUBNETMASK: 255.255.255.0.
⇒ 192.168.1.100 berada dalam range IP subnet PC Arya.

Berarti, jika kedua PC terhubung dalam jaringan lokal yang sama, baik dengan kabel maupun nirkabel, kedua PC dapat terhubung, jika minimal 1 service terbuka atau dibuka oleh administrator, misalnya ICMP (ping).

Walaupun tidak ada service yang terbuka, dalam terminologi jaringan komputer, PC Arya dan Arga dikatakan terhubung dalam subnet yang sama.

13. Default Subnetmask ip addres clas c adalah?


Dalam istilah teknologi informasi, subnet mask dan IP address adalah hal yang berbeda tapi serupa. Pada subnetmask sendiri adalah pembeda antara Network ID dengan Host ID dan sebagai penentu terhadap tujuan pengiriman paket-paket data yang dikirimkan, apakah local atau remote. Kemudian, pengertian dari IP address sendiri adalah sebuah alamat pada sebuah device yang terhubung pada sebuah jaringan, kegunaannya sendiri adalah agar device dapat berkomunikasi dengan device lainnya. 

Dalam penjabarannya, kelas-kelas IP address dibagi kedalam 3 kelompok yang dibuat berdasarkan subnet mask default. Berikut adalah subnet mask default dari masing-masing IP kelas berdasarkan biner dan desimal:

- IP Address Kelas A
Subnet Mask (biner) : 11111111.00000000.00000000.00000000
Subnet Mask (desimal) : 255.0.0.0

- IP Address Kelas B
Subnet Mask (biner) : 11111111.11111111.00000000.00000000
Subnet Mask (desimal) : 255.255.0.0

- IP Address Kelas C
Subnet Mask (biner) : 11111111.11111111.11111111.00000000
Subnet Mask (desimal) : 255.255.255.0

Kesimpulan:
Default subnet mask untuk IP address kelas C adalah 255.255.255.0

17. Berapa jumlah host maximum yang dapat digunakan pada suatu network yang memiliki subnetmask prefix /23​


Jawaban:

Jumlah host maximum yang dapat digunakan pada suatu network yang memiliki subnetmask prefix /23 adalah 510.

Ini dikarenakan subnetmask /23 mengalokasikan 23 bit untuk network dan 9 bit untuk host. Dengan 9 bit untuk host, formula yang digunakan untuk menghitung jumlah host adalah 2^9-2= 510.

Dalam rumus ini, 2^9 menghasilkan 512 (jumlah kombinasi bit yang dapat digunakan untuk host), lalu dikurangi 2 (alamat network dan broadcast) untuk mendapatkan jumlah host yang dapat digunakan.

Penjelasan:

Subnetmask merupakan bagian dari alamat IP yang digunakan untuk mengidentifikasi network dan host. Subnetmask menggunakan bit 1 untuk menunjukkan bagian network dan bit 0 untuk menunjukkan bagian host.

Subnetmask prefix /23 menunjukkan bahwa 23 bit pertama dari 32 bit alamat IP digunakan untuk menentukan network, sedangkan 9 bit sisanya digunakan untuk host.

Dalam binary, subnetmask /23 adalah 11111111.11111111.11111110.00000000. Jumlah 1 pada tiga oktet pertama (24 bit) menunjukkan bagian network, sedangkan 0 pada oktet keempat menunjukkan bagian host.

Dengan 9 bit yang dialokasikan untuk host, terdapat 2^9 = 512 kombinasi yang mungkin. Namun, dalam setiap network, dua alamat khusus, yaitu alamat network (dengan semua bit host = 0) dan alamat broadcast (dengan semua bit host = 1) tidak dapat digunakan sebagai alamat host.

Sehingga, jumlah host yang dapat digunakan adalah 2^9 - 2 = 510. Jadi, pada network dengan subnetmask prefix /23, dapat digunakan hingga 510 host.

19. Address 192.168.1.0/27 tentukan : subnetmask​


Subnetmask untuk alamat jaringan 192.168.1.0/27 adalah 255.255.255.224.

penjelasan :

Pada notasi CIDR, "/27" menunjukkan bahwa 27 bit pertama dari alamat IP tersebut digunakan untuk mengidentifikasi jaringan, sehingga 32-27=5 bit tersisa untuk mengidentifikasi host. Karena terdapat 5 bit host, maka akan ada 2^5-2=30 host yang dapat digunakan pada subnet tersebut (2^5 = 32, dikurangi 2 untuk alamat jaringan dan broadcast).

Subnetmask untuk alamat jaringan dihitung dengan menetapkan semua bit jaringan menjadi 1 dan semua bit host menjadi 0 pada sebuah bilangan biner 32-bit, yang kemudian diubah ke dalam notasi desimal dengan format empat oktet (8-bit) yang dipisahkan oleh tanda titik. Dalam hal ini, 27 bit pertama diatur menjadi 1, sedangkan 5 bit terakhir diatur menjadi 0, sehingga menghasilkan subnetmask 255.255.255.224.

25. Berapa subnetmask dari binary 11111111.11111111.11110000.00000000


Subnetmask dari binary 11111111.11111111.11110000.00000000 adalah 255.255.240.0 dengan CIDR /20.

Penjelasan:

Penentu porsi Network-id dan Host-Id pada deretan angka binner disebut dengan subnetmask. Cara menentukan subnetmask sendiri adalah dengan menghitung jumlah angka 1 dari binary terlebih dahulu. Dari binary 11111111.11111111.11110000.00000000, terdapat 20 angka 1. Jadi, nilai CIDR-nya adalah /20. Untuk CIDR /20, nilai subnetmasknya adalah 255.255.240.0. 35. jika sebuah subnetmask 255.255.255.0 dan dibagi menjadi /26,berapakah jumlah network yang terbentuk?​

Network       : 10.10.10.0

Netmask      : 255.255.255.192

Valid IP        : 10.10.10.1 - 10.10.10.62

Broadcast    : 10.10.10.63

[Network-II]

Network       : 10.10.10.64

Netmask      : 255.255.255.192

Valid IP        : 10.10.10.65 - 10.10.10.126

Broadcast    : 10.10.10.127

[Network-III]

Network       : 10.10.10.128

Netmask      : 255.255.255.192

Valid IP        : 10.10.10.129 - 10.10.10.190

Broadcast    : 10.10.10.191

[Network-IV]

Network       : 10.10.10.192

Netmask      : 255.255.255.192

Valid IP        : 10.10.10.193 - 10.10.10.254

Broadcast    : 10.10.10.255

Jadi dengan subnet /24 kita dapat membuat 4 buah network dengan /26

Anda mencoba untuk memilih partisi FAT32 atau jenis lain dari partisi yang tidak kompatibel dengan Windows Vista, Windows 7 atau Windows Server 2008 R2.

Pemecahan masalah

Untuk memecahkan masalah ini, gunakan salah satu atau lebih metode berikut.

Metode 1: Memverifikasi bahwa partisi yang kompatibel dengan Windows

Anda tidak dapat menginstal Windows di partisi FAT32. Selain itu, Anda harus benar mengkonfigurasi disk dinamis untuk digunakan dengan Windows. Untuk memverifikasi bahwa partisi kompatibel dengan Windows Vista, Windows 7 atau Windows Server 2008 R2, ikuti langkah-langkah berikut:

Untuk disk dinamis yang memiliki volume sederhana, menggunakan utilitas Diskpart.exe untuk mengkonfigurasi disk sebagai cakram yang aktif. Untuk informasi selengkapnya tentang cara menggunakan utilitas Diskpart.exe, klik nomor artikel berikut ini untuk melihat artikel di Pangkalan Pengetahuan Microsoft:

300415 penjelasan tentang utilitas baris perintah Diskpart.exe

Untuk FAT32 partisi, memformat partisi, atau mengubah partisi untuk partisi sistem file NTFS dengan menggunakan perintah Convert.exe .

Catatan Saat Anda memformat partisi, semua data yang dihapus dari partisi. Data ini mencakup semua berkas di partisi. Untuk informasi selengkapnya tentang cara menggunakan perintah Convert.exe, klik nomor artikel berikut ini untuk melihat artikel di Pangkalan Pengetahuan Microsoft:

314097 bagaimana cara menggunakan Convert.exe untuk mengubah partisi ke sistem berkas NTFS
