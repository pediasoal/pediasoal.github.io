2. bagaimana cara kerja dns dan dns server


DNS bekerja dengan menggunakan protokol UDP.Tujuanya adalah untuk performance.DNS pada saat client melakukan request terhadap DNS tujuan, yang pertama dilakukan mengecek pada komputer apakah request sudah ada di cache atau belum, jika ada buka, jika tidak kirim ke server, yang dikirimkan oleh Client, menggunakan query yang dikirim, setelah itu akan ada balasan dan diakses oleh browser.

Pada saat DNS Server menerima request, akan mencari domain (resolve) terhadap cache nya, jika sudah ada maka kembalikan ke client, jika tidak ada dicari ke server lain, namun jika pada detik tertentu, akan kemungkinan browser menyatakan timeout karena server terlalu lama merespon.

14. apa perbedaan web server, DNS server dan FTP server ?


Jawaban:

Web Server

Web Server dapat merujuk baik pada perangkat keras ataupun perangkat lunak yang menyediakan layanan akses kepada pengguna melalui protokol komunikasi HTTP atau HTTPS atas berkas-berkas yang terdapat pada suatu situs web dalam layanan ke pengguna dengan menggunakan aplikasi tertentu seperti peramban web.

DNS Server

DNS (Domain Name System) sendiri adalah sebuah sistem yang menyimpan informasi tentang nama host ataupun nama domain dalam bentuk basis data tersebar (distributed database) di dalam jaringan komputer, misalkan: Internet. DNS menyediakan alamat IP untuk setiap nama host dan mendata setiap server transmisi surat (mail exchange server) yang menerima surel (email) untuk setiap domain. Menurut browser Google Chrome, DNS adalah layanan jaringan yang menerjemahkan nama situs web menjadi alamat internet.

FTP Server

FTP Server adalah protokol internet yang digunakan untuk urusan pengiriman data dalam jaringan komputer, seperti upload dan download file yang dilakukan oleh FTP client dan FTP server.

19. bagaimana cara paling mudah buat setting DNS server?


Perangkat lunak server nama yang paling umum, disebut bernama ("nama dee") disertakan dengan BIND, Domain Nama Internet Berkeley, yang juga mencakup pustaka penyelesai dan alat lainnya. Seperti tulisan ini BIND hingga versi 9.4.2. Versi 8.3.1 juga digunakan secara luas.

Alternatif utama untuk BIND adalah djbdns, sebuah paket dari pencipta qmail. Ini dirancang untuk modularitas dan keamanan (penulis memiliki tawaran tetap $ 500 kepada siapa saja yang menemukan lubang keamanan dalam perangkat lunak, yang telah tidak diklaim sejauh ini). Ini lebih kecil dan lebih cepat dari BIND, tetapi lisensi mengendalikan kode sumber lebih ketat, dan djbdns mendapat banyak kritik untuk itu. Ada perdebatan yang memikat tentang keutamaan kedua sistem tersebut dalam arsip milis pengguna BIND.

Dalam tutorial ini, saya berbicara banyak tentang BIND, meskipun saya benar-benar menganjurkan djbdns sebagai alternatif. Berikut ini panduan untuk beralih dari BIND ke djbdns.

Agaknya Anda memiliki, paling tidak, nama domain dan alamat IP, dan Anda ingin yang menunjuk ke yang lain. Jika Anda belum mencapai titik itu, Anda mungkin ingin berbicara dengan ISP untuk melihat tentang mendapatkan alamat IP, dan registrar untuk nama domain (Jika Anda butuh bantuan, konsultasikan dengan kami Pilih dan Daftarkan Domain Nametutorial).

Pertama lihat apakah Anda sudah MENGGUNAKAN BINDING pada sistem Anda. Mungkin ada di sana, di belakang layar yang tidak terduga. Ketik bernama -v pada baris perintah. Jika mengembalikan respons yang memberi tahu Anda versi BIND yang Anda jalankan, itu berarti Anda memilikinya di sistem Anda, dan tidak perlu menginstalnya - meskipun BIND adalah target utama serangan, dan jika Anda tidak memiliki versi terbaru Anda mungkin membiarkan diri Anda terbuka terhadap risiko keamanan.

Jika tidak, unduh sumber dan unzip, lalu instal dengan ./configure; buat; buat instal. Instruksi lengkap tentang konfigurasi build untuk sistem Anda tersedia dalam file INSTALL; di sini adalah langkah-langkah untuk Mac OS X.

Setelah menginstal BIND, Anda harus mengonfigurasinya. Kami tidak memiliki ruang di sini untuk masuk ke semua perubahan konfigurasi BIND. Ada seni untuk menyempurnakan segalanya sesuai keinginan Anda - heck, ada satu buku utuh tentang topik itu.

DNS howto akan memberi tahu Anda cara mengatur file named.conf, yang berisi informasi konfigurasi untuk server nama. File-file data zona yang berisi catatan DNS yang kita bahas - biasanya, satu untuk penggunaan lokal dan satu lagi untuk penggunaan eksternal - ditempatkan dalam direktori yang ditentukan. Maka sistem Anda harus diberitahu untuk menggunakan server nama yang Anda buat. Bagaimanapun, cara-cara di tldp.org menjelaskan semua itu lebih baik daripada yang saya bisa.

25. Jelaskan keterkaitan antara dns server, web server dan database server


Jawaban:

Keterkaitan diantara ketiganya adalah bahwa DNS, Web server, dan Database server sama-sama dibutuhkan dalam menghosting sebuah website yang dynamic.

Penjelasan:

DNS = Domain Name Server, berfungsi untuk mengubah alamat ip kita menjadi sebuah nama yang kita inginkan dan memudahkan orang lain dalam mengakses website kita.

Web server = Sebuah server perangkat lunak yang berfungsi untuk melayani permintaan(request) klien berupa HTTP atau HTTPS dan akan mengirimkan respon sesuai yang diminta.

Database server = Sebuah program komputer yang menyediakan layanan pengelolaan basis data dan melayani sebuah komputer yang kemudian disalurkan ke komputer lainnya menggunakan klien-server

DNS diperlukan untuk memudahkan klien dalam mengakses website kita. Web server dibutuhkan agar terjadi request dan respon antara klien dan server. Dan database dibutuhkan untuk pengelolaan data di sebuah website kita yang menjadikannya dynamic(berubah-ubah).

   -Menyediakan alamat-alamat IP secara dinamis dan konfigurasi lain.

   -Didesain untuk melayani network yang besar dan konfigurasi TCP/IP yang kompleks.

  - Memungkinkan suatu client menggunakan alamat IP yang reusable, artinya alamat IP tersebut bisa dipakai oleh client yang lain jika client tersebut tidak sedang menggunakannya (off).

   -Memungkinkan suatu client menggunakan satu alamat IP untuk jangka waktu tertentu dari server.

   -Memberikan satu alamat IP dan parameter-parameter kofigurasi lainnya kepada client, seperti DNS Server & Default Gateway.

   -Mencegah terjadinya IP Conflict

manfaat dns server

menerjemahkan nama komputer ke IP address (memetakan). Client DNS disebut dengan resolvers dan DNS server disebut dengan name servers. Resolvers atau client mengirimkan permintaan ke name server berupa queries. Name server akan memproses dengan cara mencek ke local database DNS, menghubungi name server lainnya atau akan mengirimkan “message failure” jika ternyata permintaan dari client tidak ditemukan. Proses tersebut disebut dengan Forward Lookup Query, yaitu permintaan dari client dengan cara memetakan nama komputer (host) ke IP address.

38. Bagaimana Cara konfigurasi DNS Server di OS Linux. ?​


Penjelasan:

1.Membuat domain dari server linux dengan menggunakan TLD (Top Level Domain). TLD ini hanya berfungsi pada jaringan lokal saja. Langkahnya adalah dengan mengkonfigurasi file named.conf.local pada direktori /etc/bind

2.Buatlah file forward dengan cara copy file db.local pada direktori /etc/bind.

File forward berfungsi untuk konversi dari DNS ke IP Address. 

3.Buatlah file reserve dengan cara copy file db.127 pada direktori /etc/bind. File reserve berfungsi untuk konversi IP Address ke DNS. 

4. Tambahkan DNS nameserver dari server linux pada file resolv.conf.

5.Test apakah DNS Server tersebut berhasil atau tidak, dengan perintah ping maupun nslookup dari computer Localhost maupun computer client.
